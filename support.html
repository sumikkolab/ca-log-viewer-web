お問い合わせ前にご確認ください

  1. FAQ に該当する質問がないかご確認ください。

  3. 下記「主なトラブルシューティング」に該当する事象がないかご確認ください。

お問い合わせ時に含めていただきたい情報

  • 本アプリのバージョン (設定タブで確認可能)
  • OS バージョン (Windows 10 / 11、ビルド番号)
  • 事象の発生手順 (再現手順)
  • 画面表示のエラーメッセージまたはエラーコード
  • 個人情報 (UPN / IP / 利用者氏名) の貼り付けは不要です。再現に必要な範囲を超えて含めないでください。

主なトラブルシューティング

1. Microsoft Graph 認証画面が出ない / 失敗する

本アプリは委任認証 (利用者本人としてのアクセス) を使用します。次の点をご確認ください。

  • 設定タブの「ClientId」「TenantId」が正しく入力されているか。
  • Entra アプリ登録のリダイレクト URI に http://localhost がパブリック クライアントとして登録されているか。
  • ブラウザでサインイン用のポップアップがブロックされていないか。

2. 「権限不足」「Forbidden」 エラーが出る

必要な Microsoft Graph 権限が利用者に付与されていない可能性があります。

  • 必要なスコープ: AuditLog.Read.All / Policy.Read.ConditionalAccess / User.Read.All。
  • 「API のアクセス許可」画面で 3 つすべてが追加され、グローバル管理者により管理者の同意が付与されているかご確認ください。
  • 委任認証では、サインインしているユーザーにも当該スコープを付与できる役割 (Global Reader / Security Reader 等) が必要です。

3. UPN が空欄 / 「未取得」になる

UPN は Windows DPAPI で暗号化された短期キャッシュ (最大 7 日) 経由で表示されます。次の対処をお試しください。

  • 調査タブから対象行を再選択し、UPN の再取得を試す (キャッシュ失効の場合に再フェッチします)。
  • 取得タブで対象期間を再取得する (Microsoft Graph から最新の UPN 解決を実行します)。
  • 同時に「権限不足」エラーが出ている場合は、上記 2 の手順をご確認ください。

4. ポリシー未適用 (Not applied) のサインインが多い

これは多くの場合、不具合ではなく仕様どおりの動作です。詳細は FAQ の「ポリシー未適用 (Not applied) のサインインが多く見えます」をご参照ください。

5. アプリが起動しない / 起動直後に終了する

  • Windows のバージョンが 10 (build 19041 以降) または 11 であることをご確認ください。
  • Microsoft Store から再インストールしてください。
  • 事象が継続する場合は、上記「お問い合わせ時に含めていただきたい情報」を添えて連絡先までご連絡ください。

Before contacting us

  1. Please check whether your question is covered in the FAQ.
  2. Please review the steps in Getting Started (Entra app registration / Microsoft Graph permission grant).
  3. Please review the "Common troubleshooting" section below for matching symptoms.

Information to include in your inquiry

  • App version (visible in the Settings tab)
  • OS version (Windows 10 / 11, build number)
  • Steps to reproduce the issue
  • Error messages or codes shown on screen
  • You do not need to include personal information (UPN / IP / user name). Please do not include data beyond what is needed to reproduce the issue.

Common troubleshooting

1. Microsoft Graph sign-in does not appear / fails

The App uses delegated authentication (access as the signed-in user). Please verify the following.

  • "ClientId" and "TenantId" are entered correctly in the Settings tab.
  • The Entra app registration has http://localhost registered as a public client redirect URI.
  • Sign-in popups are not blocked by the browser.

2. "Insufficient permissions" or "Forbidden" error

The signed-in user may lack the required Microsoft Graph permissions.

  • Required scopes: AuditLog.Read.All / Policy.Read.ConditionalAccess / User.Read.All.
  • Confirm in the "API permissions" screen that all three are added and that admin consent has been granted by a Global Administrator.
  • With delegated authentication, the signed-in user also needs a role (Global Reader / Security Reader, etc.) that allows these scopes.

3. UPN is empty / shown as "not resolved"

UPNs are displayed via a short-term cache encrypted with Windows DPAPI (up to 7 days). Try the following.

  • Re-select the target row in the Investigate tab to retry UPN resolution (re-fetched if the cache expired).
  • Re-fetch the target range from the Fetch tab (resolves the latest UPNs from Microsoft Graph).
  • If you also see an "Insufficient permissions" error, please follow step 2 above.

4. Many sign-ins show as "Not applied"

This is often by design and not a defect. See the FAQ entry "Many sign-ins look 'Not applied'" for details.

5. The App does not start / exits immediately

  • Confirm Windows is version 10 (build 19041 or later) or 11.
  • Reinstall from the Microsoft Store.
  • If the issue persists, please contact us with the items listed in "Information to include in your inquiry" above.

If you discover a security concern, please contact us by email (support@sumikkolab.com) rather than via public channels. Including "Security" in the subject helps us prioritize. Please wait for our response before disclosing details publicly.
