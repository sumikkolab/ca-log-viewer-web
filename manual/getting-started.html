一本道: 初回起動 → 取得 → 分析 → レポート

Step 1. 初回起動

  1. Microsoft Store からインストール (または GitHub Releases から MSIX ダウンロード)
  2. 初回起動時に利用規約・プライバシー方針への同意 (Phase 19 Step 4 で実装予定)
  3. 言語選択 (日本語 / English) — 設定タブから後で変更可能

Step 2. 取得 (Microsoft Graph 経由)

  1. Entra アプリ登録 (Azure portal でアプリ作成、ClientId 取得)
  2. 必要な Graph 権限を同意: AuditLog.Read.All / Policy.Read.ConditionalAccess / User.Read.All
  3. アプリで「取得」ボタン → サインインログ + CA ポリシーを取得

Step 3. 分析 (調査タブ)

  1. 統一フィルタ (期間 / ユーザー / CA 状態 / 判定 / デバイス / 特殊) で絞り込み
  2. 一覧から行を選択 → 詳細パネルで 7 ブロックを確認 (結論 / サインイン概要 / ユーザー / アプリ / 条件付きアクセス / 適用ポリシー / 端末 / 許可拒否理由)
  3. 傾向タブで異常検知結果 / アプリ別集計を確認

Step 4. レポート (HTML 監査レポート出力)

  1. 調査タブ上部「Export HTML」ボタン → 現在のフィルタ結果を HTML レポートとして出力
  2. ブラウザで自動オープン → Summary / Assessment / 集約表 (ユーザー別 / アプリ別 / ポリシー別) / 観点別 / Allow & Deny reasons を確認
  3. 外部共有時は UPN 含むため取り扱いに注意

Linear flow: First launch → Fetch → Analyze → Report

Step 1. First launch

  1. Install from Microsoft Store (or download MSIX from GitHub Releases)
  2. Accept Terms / Privacy on first launch (planned for Phase 19 Step 4)
  3. Choose language (Japanese / English) — switchable later from Settings

Step 2. Fetch (via Microsoft Graph)

  1. Register an Entra app (Azure portal, retrieve ClientId)
  2. Grant required Graph permissions: AuditLog.Read.All / Policy.Read.ConditionalAccess / User.Read.All
  3. Click "Fetch" in the app → retrieves sign-in logs + CA policies

Step 3. Analyze (Investigate tab)

  1. Filter by unified filters (Time range / User / CA status / Judgment / Device / Special)
  2. Select a row → review 7 blocks in the Detail Pane (Conclusion / Sign-in summary / User / App / Conditional Access / Applied policies / Device / Allow & Deny reasons)
  3. Check anomaly detection / per-app aggregation in the Trends tab

Step 4. Report (Export HTML audit report)

  1. Click "Export HTML" at the top of the Investigate tab → exports the current filter result as an HTML report
  2. Opens automatically in the browser → review Summary / Assessment / Aggregation (by user / app / policy) / aspect tables / Allow & Deny reasons
  3. Be careful when sharing externally — the report includes UPNs
