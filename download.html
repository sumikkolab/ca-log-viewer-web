CA Log Viewer は以下の経路で入手できます。Phase 19 Step 5 (Microsoft Store 提出) の完了までは、すべて「準備中」表示になります。

入手経路

Microsoft Store 推奨準備中

もっとも安全で簡単な入手経路です。自動更新に対応します。

  • 署名済み MSIX を Microsoft が配信
  • Windows Update 経由の自動更新
  • デモモードは含まれません
Microsoft Store で開く (Phase 19 Step 5 で公開)

GitHub Releases (MSIX) 準備中

Store 公開と同等のビルドに加えてデモモードを含みます。Store 経由が利用できない環境向け。

  • 署名済み MSIX (Store と同等)
  • デモモード同梱 (110 件 / 6 シナリオ)
  • 手動更新 (新バージョンの MSIX を再インストール)
GitHub Releases (Phase 19 Step 5 で公開)

GitHub リポジトリ 準備中

本サイト (ca-log-viewer-web) のソースのみ公開予定です。アプリ本体のソースは非公開です。

  • 製品ページ / マニュアル / FAQ などのソース
  • Issue を起票してフィードバック可能 (Phase 19 Step 5 以降)
GitHub で見る (Phase 19 Step 5 で公開)

動作環境

  • OS: Windows 10 (build 19041 以降) / Windows 11
  • .NET 8 ランタイム (MSIX に同梱)
  • ネットワーク: Microsoft Graph (graph.microsoft.com) への HTTPS アクセス
  • Entra ID テナント (実データ利用時)

必要な Microsoft Graph 権限

  • AuditLog.Read.All (サインインログ取得)
  • Policy.Read.ConditionalAccess (CA ポリシー定義取得)
  • User.Read.All (UPN / 表示名解決)
委任認証のみを使用します。アプリケーション認証 (クライアントシークレット / 証明書) は使用しません。詳細は はじめに を参照してください。

System requirements

  • OS: Windows 10 (build 19041 or later) / Windows 11
  • .NET 8 runtime (bundled in the MSIX)
  • Network: HTTPS access to Microsoft Graph (graph.microsoft.com)
  • Entra ID tenant (when using real data)

Required Microsoft Graph permissions

  • AuditLog.Read.All (read sign-in logs)
  • Policy.Read.ConditionalAccess (read CA policy definitions)
  • User.Read.All (resolve UPN / display name)
Only delegated authentication is used. Application authentication (client secret / certificate) is not used. See Getting Started for details.
